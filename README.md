# Steward Landing Page

> Everything your restaurant runs on. One quiet system.

---

## Overview

This repository contains the complete design documentation and implementation of the official Steward landing page.

The objective is not simply to build a marketing website, but to communicate Steward's product philosophy, operational approach, and value to restaurant owners through a calm, trustworthy, and editorial experience.

This repository follows a documentation-first workflow. Every decision is specified before implementation begins.

---

## About Steward

Steward is a restaurant operating system built to help restaurant owners spend less time managing operations and more time running their business.

It brings together ordering, kitchen operations, payments, and daily operational insights into one dependable system.

Steward is designed around a simple principle:

> Give restaurant owners their time back.

Technology is never the hero.

The restaurant owner is.

---

## Repository Purpose

This repository exists to:

- Define the visual and product direction of the Steward landing page.
- Document every design and UX decision before development.
- Maintain consistency between design, engineering, and product.
- Serve as the single source of truth throughout implementation.

## Homepage Structure

The homepage follows a narrative rather than a feature-driven approach.

1. Navigation
2. Hero
3. Needle
4. Dashboard Preview
5. One System
6. A Day with Steward
7. Built for the Way Restaurants Actually Work
8. Pricing
9. Early Partners
10. FAQ
11. Final Call to Action
12. Footer

Each section answers a single user question and contributes to the overall story of the product.

---

## Guiding Principles

Throughout this project:

- Build trust before selling.
- Explain before persuading.
- Stay truthful to current product capabilities.
- Avoid exaggerated marketing language.
- Design for clarity rather than attention.
- Every interface element must have a purpose.

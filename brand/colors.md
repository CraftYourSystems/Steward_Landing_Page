# Purpose

Color is used to communicate clarity, hierarchy, and operational state.

Steward avoids decorative color usage.

Every color should have a clear purpose.

Whitespace remains the primary design element.

Color supports the interface—it never becomes the interface.

---

# Color Philosophy

Restaurants are busy environments.

The software should reduce visual noise rather than contribute to it.

Colors should feel:

- Calm
- Dependable
- Editorial
- Professional
- Timeless

Avoid colors that feel:

- Loud
- Playful
- Futuristic
- Neon
- Overly saturated

Steward should remain visually quiet.

---

# Primary Brand Color

Purpose

The primary brand color represents Steward itself.

Used sparingly for:

- Primary actions
- Active navigation
- Brand highlights
- Important links

The primary color should never dominate the interface.

# Neutral Palette

The neutral palette forms the majority of the interface.

Backgrounds, cards, borders, typography, and spacing rely primarily on neutral colors.

Suggested hierarchy:

Background

Surface

Elevated Surface

Border

Muted Border

Primary Text

Secondary Text

Muted Text

Disabled Text

Each neutral should have sufficient contrast while maintaining a calm appearance.

---

# Semantic Colors

Semantic colors communicate operational state.

They should never be used decoratively.

## Success

Used for:

- Completed actions
- Connected systems
- Operational health

Should communicate reassurance.

---

## Warning

Used for:

- Attention required
- Minor operational issues

Should communicate awareness rather than urgency.

---

## Error

Used for:

- Failed operations
- Critical issues

Should remain calm.

Avoid aggressive reds.

---

## Information

Used for:

- Neutral operational updates
- Status messages
- Informational notices

Should remain understated.

# Border Colors

Borders should separate—not decorate.

Use subtle contrast.

Avoid heavy outlines.

Cards should feel defined through spacing before borders.

---

# Interactive Colors

Hover

---

Accessible.

# Operational Principles

Color should never communicate information by itself.

Every color-coded state must include:

- Text
- Icon
- Position
- Context

This ensures accessibility.

---

# Dashboard Principles

Dashboard colors should prioritize readability.

Charts should remain restrained.

Avoid rainbow palettes.

Use semantic colors consistently.

Large areas of saturated color should be avoided.

---

# Accessibility

All text/background combinations must meet WCAG AA contrast requirements at minimum.

Interactive elements must remain distinguishable without relying only on color.

Reduced vision users should understand every interface state.

---

# Do

✓ Use whitespace first.

✓ Use color intentionally.

✓ Keep semantic meanings consistent.

✓ Prefer neutral surfaces.

✓ Let typography create hierarchy.

---

# Don't

✗ Use gradients as decoration.

✗ Use neon colors.

✗ Use unnecessary accent colors.

✗ Introduce colors without semantic purpose.

✗ Allow color to overpower content.

# Purpose

The Steward logo is the visual signature of the brand.

It should communicate confidence, simplicity, and longevity.

The logo should never become decorative.

It exists to identify Steward—not to dominate the interface.

---

# Logo Philosophy

A good logo should feel:

- Quiet
- Memorable
- Timeless
- Professional

It should still feel appropriate five years from now.

Avoid trends that may quickly become dated.

The logo should represent operational confidence rather than technological novelty.

---

# Primary Logo

The primary logo consists of:

- Steward wordmark
- Steward mark (symbol)

This is the preferred version across all digital experiences.

---

# Secondary Logo

The secondary logo may include:

- Symbol only
- Wordmark only

Use only where space is limited.

Examples:

- Mobile navigation
- Browser favicon
- App icon
- Social profile picture

---

# Logo Variants

## Full Color

Use on light backgrounds whenever possible.

Preferred version.

---

## Light Version

Use only on dark backgrounds.

Maintain sufficient contrast.

---

## Monochrome

Use only when color reproduction is unavailable.

Example:

- Documents
- Print
- Embossing
- Merchandise

---

# Minimum Size

The logo must remain legible at all sizes.

### Wordmark

Minimum width:

120px

---

### Symbol

Minimum width:

24px

---

### Favicon

16×16

32×32

48×48

SVG preferred where supported.

---

# Clear Space

Maintain sufficient whitespace around the logo.

Minimum clear space:

Height of the "S" in Steward.

No text or graphics should enter this space.

The logo should never appear crowded.

---

# Placement

Preferred positions:

- Top-left navigation
- Footer
- Documentation cover pages

Avoid placing the logo in visually busy areas.

Allow surrounding whitespace to reinforce the brand.

---

# Background Usage

### Light Backgrounds

Use the standard logo.

---

### Dark Backgrounds

Use the light variant.

---

### Photography

Only if sufficient contrast exists.

If readability is reduced, introduce a subtle overlay rather than altering the logo.

---

# Scaling

Always scale proportionally.

Never stretch.

Never compress.

Never distort.

Use vector assets whenever possible.

Preferred format:

SVG

---

# Color Usage

Use only approved brand colors.

Never recolor the logo for decoration.

Do not introduce gradients, patterns, or textures inside the mark.

---

# Incorrect Usage

Never:

✗ Stretch the logo

✗ Rotate the logo

✗ Change proportions

✗ Apply drop shadows

✗ Add outlines

✗ Apply gradients

✗ Use glow effects

✗ Place on low-contrast backgrounds

✗ Place inside random shapes

✗ Change typography

✗ Rearrange the symbol and wordmark

Consistency builds recognition.

---

# Favicon

The favicon should use:

Steward symbol only.

No wordmark.

Designed for clarity at very small sizes.

Provide:

- SVG
- PNG (16px)
- PNG (32px)
- PNG (48px)
- Apple Touch Icon (180×180)

---

# App Icon

The app icon should:

- Use the Steward symbol
- Maintain generous padding
- Avoid tiny details
- Be recognizable at small sizes

Do not include the wordmark.

# Naming Convention

Use descriptive filenames.

Examples:

steward-logo-primary-light.svg

steward-logo-primary-dark.svg

steward-symbol.svg

steward-wordmark.svg

steward-favicon-32.png

Avoid ambiguous names like:

logo-final.png

logo-new.svg

final-final-logo.png

---

# Accessibility

The logo should always maintain sufficient contrast against its background.

It should never be the only way users identify Steward.

Text-based identification should always be available where appropriate.

# Guiding Principle

The Steward logo should feel as dependable as the product it represents.

Recognition comes from consistency—not repetition.

Protect the logo by using it with restraint, respecting its space, and allowing the product experience to carry the brand.

# Homepage Specification

## 04 — Dashboard Preview

Version: 1.0

Status: Ready for Implementation

References

- Steward Constitution
- Homepage Architecture
- Needle Philosophy

---

# Purpose

The Dashboard Preview provides evidence that Steward is a real operational platform.

It should reinforce the trust established by the Needle section by showing that the operational briefing is generated from a real system.

The objective is not to demonstrate every feature.

The objective is to validate the product.

---

# User Question

"Is this an actual product?"

---

# Business Goal

- Build credibility.
- Demonstrate product maturity.
- Connect Needle to the Steward platform.
- Encourage visitors to continue exploring.

---

# Core Message

Needle is powered by a real operational system.

Every insight, report, and operational update comes from Steward's connected workflows.

---

# Content

## Eyebrow

Inside Steward

---

## Heading

Operational clarity, built into every service.

---

## Supporting Copy

Every update from Needle is grounded in real restaurant activity.

Orders, kitchen operations, payments, and daily reporting stay connected in one dependable operating system.

---

# Primary Component

Dashboard Preview

Only the upper portion of the dashboard should be displayed.

The preview should intentionally stop before exposing detailed functionality.

Its purpose is to create confidence, not replace a product demonstration.

---

# Dashboard Content

Visible Elements

- Welcome header
- Operational status summary
- Today's overview
- Active modules
- Simple operational cards
- Navigation sidebar

Avoid

- Large analytics charts
- Marketing metrics
- Dense tables
- Complex reports
- Administrative settings
- Fake activity feeds

The preview should feel clean and purposeful.

---

# Layout

Centered.

Large dashboard image below supporting copy.

Maximum width aligned with the page container.

Enough surrounding whitespace to make the dashboard the focal point.

---

# Visual Hierarchy

1. Dashboard Preview

2. Heading

3. Supporting Copy

The dashboard itself is the hero of this section.

Nothing else should compete visually.

---

# Visual Style

Dark graphite interface.

Soft elevation.

Rounded corners.

Subtle borders.

Premium appearance.

Minimal visual noise.

Realistic proportions.

No exaggerated UI effects.

---

# Motion

Dashboard fades into view.

Very small upward movement.

Optional subtle parallax while scrolling.

No animated charts.

No counting numbers.

No flashing indicators.

No fake live updates.

The interface should appear calm and stable.

---

# Interaction

Desktop

Slight hover elevation.

Optional gentle zoom under 2%.

Mobile

Static preview.

No interaction required.

The screenshot is not a product demo.

---

# Accessibility

Meaningful alt text.

Responsive scaling.

Keyboard users should not miss any essential information.

The screenshot should not contain information required to understand the product.

---

# Copy Principles

The copy should reinforce:

Trust.

Operational clarity.

Connected workflows.

Never discuss individual features in detail.

Never oversell the dashboard.

---

# Engineering Notes

Use an optimized image or rendered mockup.

Lazy load the preview.

Preserve aspect ratio.

Responsive scaling.

Future replacement with production dashboard should require no layout changes.

---

# Acceptance Criteria

Visitors should understand:

- Steward is a real platform.
- Needle comes from actual operational data.
- The dashboard supports the product rather than being the product.
- The product appears mature and dependable.

The preview should increase confidence without overwhelming visitors.

---

# Out of Scope

Interactive dashboard.

Feature walkthrough.

Clickable widgets.

Animated metrics.

Live data.

Charts.

Complex analytics.

Settings screens.

Product tour.

These belong inside the product or demo experience.

---

# Definition of Done

The Dashboard Preview is complete when visitors naturally conclude:

"Needle isn't making things up.

It's reporting from a real operational system."

The preview should feel like evidence, not advertising.

# Homepage Specification

## 02 — Hero

Version: 1.0

Status: Ready for Implementation

References

- Steward Constitution
- Homepage Architecture

---

# Purpose

The Hero introduces Steward in one clear idea.

It should immediately answer what Steward is and why it exists.

The Hero is not a sales pitch.

It is the opening statement of the product.

Visitors should understand Steward before scrolling.

---

# User Question

"What is Steward?"

---

# Business Goal

- Introduce Steward clearly.
- Establish product positioning.
- Communicate the primary value proposition.
- Encourage visitors to continue exploring.

---

# Core Message

Steward brings every essential restaurant operation into one dependable operating system.

The restaurant owner remains the hero.

Steward quietly supports daily operations.

---

# Content

## Eyebrow

Restaurant Operating System

---

## Headline

Everything your restaurant runs on.

One quiet system.

---

## Supporting Copy

Steward brings ordering, payments, kitchen operations, and daily insights into one dependable system—so you spend less time managing and more time running your restaurant.

---

## Primary CTA

Book a Demo

---

## Secondary CTA

Explore Steward

Scrolls smoothly to the Needle section.

---

# Layout

Centered content.

Maximum width approximately 700–800px.

Content aligned vertically.

Large headline.

Supporting paragraph below.

CTA group below paragraph.

Needle section should be visible or nearly visible without requiring a full scroll on common desktop screens.

---

# Visual Hierarchy

1. Headline

2. Supporting Copy

3. Primary CTA

4. Secondary CTA

Nothing else should compete for attention.

---

# Visual Style

Minimal.

Editorial.

Large typography.

Generous whitespace.

No decorative graphics.

No marketing illustrations.

No floating cards.

No background patterns.

The typography should carry the visual impact.

---

# Background

Simple.

Bright.

Minimal texture if any.

No gradients competing with content.

The Hero should transition naturally into the Needle section.

---

# Motion

Headline fades into view.

Supporting copy appears shortly after.

CTA buttons appear last.

Animations should be subtle.

No dramatic entrance effects.

No scaling.

No bouncing.

No rotating.

---

# Interaction

Primary CTA

Navigates to demo booking.

Secondary CTA

Smooth scroll to Needle.

Hover

Small opacity adjustment.

Very subtle elevation if appropriate.

Focus

Accessible keyboard focus.

---

# Responsive Behavior

Desktop

Centered.

Large typography.

Balanced whitespace.

Tablet

Reduced font sizes.

Adjusted spacing.

Mobile

Single-column layout.

Buttons stacked vertically.

Readable line lengths.

Comfortable touch targets.

---

# Accessibility

Semantic heading structure.

Single H1.

Buttons fully keyboard accessible.

Sufficient color contrast.

Readable typography.

No information conveyed only through visuals.

---

# Copy Principles

The Hero should:

Explain.

Not persuade.

State facts.

Avoid hype.

Avoid technical jargon.

Avoid buzzwords.

Avoid AI terminology.

---

# Engineering Notes

Responsive typography.

Responsive spacing.

Fluid layout.

Lazy-load below-the-fold assets.

Maintain excellent Largest Contentful Paint (LCP).

---

# Acceptance Criteria

The Hero should:

✓ Explain Steward within a few seconds.

✓ Clearly communicate the product category.

✓ Present one primary value proposition.

✓ Guide visitors naturally toward Needle.

✓ Feel calm and premium.

✓ Avoid visual clutter.

✓ Perform well across all screen sizes.

---

# Out of Scope

Feature comparison tables.

Customer logos.

Animated dashboards.

Statistics.

Trust badges.

Videos.

Product walkthroughs.

Background illustrations.

These belong elsewhere on the homepage.

---

# Definition of Done

The Hero is complete when a first-time visitor can answer:

- What is Steward?
- Who is it for?
- Why should I continue scrolling?

without needing additional explanation.

The Hero should create understanding—not excitement.
